# 07: Repo Brain And Memory Projection

## Goal

Add a reviewable repo brain as a projection from mission events, handoffs, reviews, and accepted promotion candidates.

## Why

Presence needs continuity, but memory must not become orchestration truth. The repo brain should be inspectable, diffable, and evidence-backed.

## Outcomes

- Knowledge pages follow compiled-truth plus timeline.
- Promotion candidates require review before becoming durable knowledge.
- Memory records link to tickets, attempts, mission events, files, commands, or review artifacts.
- Memory can be read-only, read-write, or denied per repo.
